Reference for integrating with the TatvaCare CRM — the single source of truth for patient leads across every TatvaCare product line.

## What this API does

Write leads (create / update / upsert). Read leads. Discover the data model. Attach files. That's it. Sales operations happen inside the CRM UI — this API is for **systems** putting leads INTO the CRM and reading them back.

## The shape

```
 CRM Lead (one row per patient — deduped by mobile_no)
   ├── Plan Profile           — what plan / PSP / insurance / drug program / subscription
   ├── Lab Profile            — lab values, vitals, conditions, scores
   └── Care & Providers       — coaches, RM, therapist, doctor, hospital
```

The lead carries identity, contact, and routing fields. The three children carry concern-specific data. A patient typically has 1–3 child rows populated. Multi-program is normal.
